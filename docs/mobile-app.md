# VirtuaFC Mobile App — Build & Publish Guide

This document covers everything needed to build, test, and publish the VirtuaFC native mobile apps for iOS (App Store) and Android (Google Play). The apps are thin Capacitor wrappers that load the production web app inside a native WebView.

## Architecture

```
┌──────────────────────────────────┐
│  Native App Shell (Capacitor)    │
│  iOS: WKWebView                  │
│  Android: Android WebView        │
├──────────────────────────────────┤
│  Native Plugins                  │
│  Push notifications, haptics,    │
│  status bar, splash screen,      │
│  back button (Android)           │
├──────────────────────────────────┤
│  WebView loads remote URL        │
│  https://virtuafc.com            │
└──────────────────────────────────┘
```

The app is **not** a bundled SPA — it loads the live server-side rendered Laravel app from `https://virtuafc.com`. This means:

- Web content updates deploy instantly (no app store update needed)
- Session cookies are managed natively by the WebView
- CSRF tokens work normally via Blade `@csrf` directives
- The app requires an internet connection to function

Native app updates are only needed when Capacitor plugins, native config, or store metadata change.

## Project Structure

```

resources/js/native-bridge.js                # JS bridge: push, haptics, status bar, back button

database/migrations/..._create_device_tokens_table.php
ios/                                         # Xcode project (auto-generated by Capacitor)
android/                                     # Android Studio project (auto-generated by Capacitor)
```

### Key Configuration

**`capacitor.config.ts`** — the main Capacitor config. Important settings:

- `server.url`: The remote URL the app loads (`https://virtuafc.com`)
- `plugins.SplashScreen`: Dark splash screen matching the app's `#0f172a` background
- `plugins.PushNotifications`: Enables badge, sound, and alert presentation
- `ios.scheme`: Custom URL scheme for deep linking
- `android.backgroundColor`: Native background color before WebView loads

### Native Bridge

**`resources/js/native-bridge.js`** detects the native environment via `Capacitor.isNativePlatform()` and activates:

- **Push notifications**: Requests permission, registers device token with the server
- **Status bar**: Dark style with `#0f172a` background
- **Haptic feedback**: Importable `triggerHaptic()` function for game events
- **Android back button**: Hardware back navigates browser history or exits app

All functions are no-ops in regular browsers — the web app works identically with or without the native bridge.

## Building the Apps

### Step 1: Sync Capacitor

After any changes to `capacitor.config.ts`, plugins, or the `public/` directory:

```bash
npx cap sync
```

This copies web assets and updates native plugin configurations in both platform projects.

### Step 2: Build for iOS

```bash
# Open the Xcode project
npx cap open ios
```

In Xcode:

1. **Select the signing team**: `App` target > Signing & Capabilities > Team
2. **Set the bundle identifier**: Should be `com.virtuafc.app`
3. **Add Push Notification capability**: `App` target > Signing & Capabilities > + Capability > Push Notifications
4. **Set deployment target**: iOS 16.0 minimum (recommended)
5. **Add `GoogleService-Info.plist`**: Drag it into `App/App/` in the Xcode project navigator
6. **Build**: Product > Build (Cmd+B)
7. **Run on simulator**: Select a simulator and press Run (Cmd+R)
8. **Archive for App Store**: Product > Archive (requires a physical device or "Any iOS Device" selected)

### Step 3: Build for Android

```bash
# Open the Android Studio project
npx cap open android
```

In Android Studio:

1. **Sync Gradle**: File > Sync Project with Gradle Files
2. **Add `google-services.json`**: Place in `android/app/` (if not already there)
3. **Set signing config**: Build > Generate Signed Bundle/APK > Create new keystore (first time only)
4. **Build debug APK**: Build > Build Bundle(s)/APK(s) > Build APK(s)
5. **Run on emulator**: Select an emulator and press Run
6. **Build release bundle**: Build > Generate Signed Bundle/APK > Android App Bundle (.aab)

#### Android Keystore

For release builds, create a signing keystore (keep it safe — you cannot change it after publishing):

```bash
keytool -genkey -v -keystore virtuafc-release.keystore \
  -alias virtuafc -keyalg RSA -keysize 2048 -validity 10000
```

Store the keystore file and credentials securely. **Never commit the keystore to git.**

## Push Notifications Setup

### How It Works

1. The native app calls `PushNotifications.register()` on first login
2. The OS returns a device token (APNs token on iOS, FCM token on Android)
3. The native bridge sends the token to `POST /api/device-token`
4. The server stores it in the `device_tokens` table, linked to the user
5. When game events happen, the server sends push notifications via FCM

### Server-Side: Sending Pushes

To send push notifications, integrate Firebase Cloud Messaging on the server. Install the Firebase PHP SDK:

```bash
composer require kreait/firebase-php
```

Add the Firebase credentials to your `.env`:

```
FIREBASE_CREDENTIALS=/path/to/firebase-service-account.json
```

Create a service to send pushes (suggested location: `app/Modules/Notification/Services/PushNotificationService.php`):

```php
use Kreait\Firebase\Factory;
use Kreait\Firebase\Messaging\CloudMessage;
use Kreait\Firebase\Messaging\Notification;

class PushNotificationService
{
    public function sendToUser(int $userId, string $title, string $body, array $data = []): void
    {
        $tokens = DeviceToken::where('user_id', $userId)->pluck('token')->all();
        if (empty($tokens)) return;

        $messaging = (new Factory)
            ->withServiceAccount(config('services.firebase.credentials'))
            ->createMessaging();

        $notification = Notification::create($title, $body);

        foreach ($tokens as $token) {
            $message = CloudMessage::withTarget('token', $token)
                ->withNotification($notification)
                ->withData($data);

            try {
                $messaging->send($message);
            } catch (\Throwable $e) {
                // Token may be invalid — remove it
                DeviceToken::where('token', $token)->delete();
            }
        }
    }
}
```

### Notification Triggers

Hook into the existing `NotificationService::create()` method to dispatch pushes for key game events:

| Notification Type | Push? | Rationale |
|-------------------|-------|-----------|
| `TYPE_MATCH_RESULT` | Yes | Users want to know match outcomes |
| `TYPE_TRANSFER_BID_RESULT` | Yes | Transfer bid accepted/rejected |
| `TYPE_SEASON_CHAMPION` | Yes | Major achievement |
| `TYPE_SEASON_PROMOTED` | Yes | Major achievement |
| `TYPE_CONTRACT_RENEWAL` | Yes | Contract decisions |
| `TYPE_INJURY` | No | Too frequent, low urgency |
| `TYPE_TRANSFER_WINDOW_OPEN` | Yes | Seasonal reminder |

### iOS-Specific: APNs Configuration

Firebase handles APNs delivery, but you need to configure the APNs key:

1. In Apple Developer portal > Certificates, Identifiers & Profiles > Keys
2. Create a new key with "Apple Push Notifications service (APNs)" enabled
3. Download the `.p8` file
4. In Firebase Console > Project Settings > Cloud Messaging > Apple app configuration
5. Upload the `.p8` file with the Key ID and Team ID

## Testing

### Local Development

For local testing, update `capacitor.config.ts` temporarily:

```typescript
server: {
  // Use your local dev server URL
  url: 'http://192.168.1.x:8000', // your machine's LAN IP
  cleartext: true, // allow HTTP for local dev
},
```

Then run:

```bash
npx cap sync
npx cap run ios       # Run on iOS simulator
npx cap run android   # Run on Android emulator
```

**Remember to revert the URL to `https://virtuafc.com` before building for production.**

### Testing Checklist

- [ ] App launches with splash screen, transitions to login/dashboard
- [ ] Login works, session persists across app backgrounding and foregrounding
- [ ] All navigation works (dashboard, squad, lineup, finances, transfers, calendar, standings)
- [ ] Match simulation runs correctly (live-match.js in WebView)
- [ ] Lineup drag-and-drop works with touch events
- [ ] Push notification permission prompt appears on first login
- [ ] Device token is registered (check `device_tokens` table)
- [ ] Receiving a push notification shows the alert
- [ ] Tapping a push notification opens the correct screen
- [ ] Android back button navigates history correctly
- [ ] iOS safe areas render properly (no content behind notch/Dynamic Island)
- [ ] Status bar has dark style with correct background color
- [ ] External links (GitHub, legal) open in system browser, not WebView
- [ ] Session expiry (419 response) redirects to login correctly
- [ ] App works on older devices (iPhone SE, budget Android phones)

### Testing Push Notifications

Without the full server-side integration, you can test pushes via the Firebase Console:

1. Go to Firebase Console > Cloud Messaging > Send your first message
2. Enter a title and body
3. Select "Send test message"
4. Paste a device token from the `device_tokens` table
5. Send — the push should appear on the device

## Updating the Apps

### Web Content Updates

Since the app loads from `https://virtuafc.com`, web content updates deploy instantly when you deploy the Laravel app. No app store update is needed.

### Native Updates (Require Store Submission)

You need a new store submission when:

- Capacitor plugins are added, removed, or updated
- `capacitor.config.ts` settings change
- Native project files change (iOS entitlements, Android permissions)
- Store metadata changes (screenshots, description)

Process:

```bash
# 1. Make changes to config/plugins
# 2. Sync to native projects
npx cap sync

# 3. Bump version numbers
#    iOS: Xcode > App target > General > Version / Build
#    Android: android/app/build.gradle > versionCode / versionName

# 4. Build and submit (same as initial submission)
```

### Version Strategy

The native app version is independent of the web app. Suggested scheme:

- **1.0.0** — Initial release
- **1.0.1** — Bug fixes (native side)
- **1.1.0** — New native plugin or feature
- **2.0.0** — Major Capacitor upgrade

## Troubleshooting

### WebView Issues

**Blank white screen on launch**: The remote server may be unreachable. Check the URL in `capacitor.config.ts` and verify the server is up.

**Session lost after backgrounding**: iOS may purge WKWebView cookies under memory pressure. Consider increasing `SESSION_LIFETIME` in `.env` (default is 120 minutes). A value of 480 (8 hours) is recommended for mobile.

**419 CSRF errors**: The existing Axios interceptor in `bootstrap.js` handles this by redirecting to login. If frequent, increase session lifetime.

**Content cut off by notch**: Verify the viewport meta tag includes `viewport-fit=cover` and the safe area CSS insets are applied in `app.css`.

### Build Issues

**iOS: "No signing certificate"**: In Xcode, select your Apple Developer team in Signing & Capabilities. You need an active Apple Developer Program membership.

**Android: "SDK not found"**: Set `ANDROID_HOME` environment variable or configure the SDK path in Android Studio > Preferences > System Settings > Android SDK.

**Capacitor sync fails**: Ensure `node_modules` is installed (`npm install`) and all Capacitor packages are at compatible versions.

### Push Notification Issues

**No permission prompt on iOS**: Push notifications require a physical device — they don't work in the iOS Simulator.

**Token not reaching server**: Check the browser console in Safari Web Inspector (iOS) or Chrome DevTools (Android). The native bridge logs warnings on failures.

**Push not received**: Verify the Firebase project has the correct APNs key (iOS) or that `google-services.json` is in the right location (Android). Check Firebase Console > Cloud Messaging for delivery reports.

## Security Considerations

- The `X-Frame-Options: DENY` header in `SecurityHeaders` middleware does **not** affect Capacitor's WebView (it's not an iframe). No changes needed.
- CSRF protection works normally — the WebView is a first-party context for the domain.
- Session cookies use `SameSite=Lax` which works correctly in Capacitor WebViews.
- The device token API endpoint (`POST /api/device-token`) is protected by `auth:web` middleware.
- **Never commit** the Android signing keystore, Firebase service account JSON, or APNs `.p8` key to git.
